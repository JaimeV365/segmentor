# Backup Guide for segmentor.app

## What to Backup

### Essential Files & Folders (Always Backup)
1. **`src/`** - All React source code, components, and TypeScript files
2. **`public/`** - Static HTML files, images, fonts, icons, and assets
3. **`scripts/`** - Build scripts and deployment automation
4. **`package.json`** - Project dependencies and scripts
5. **`package-lock.json`** - Locked dependency versions

### Configuration Files (Important)
- `.gitignore` - Git ignore rules
- `tsconfig.json` - TypeScript configuration
- `_redirects` - Cloudflare Pages redirect rules
- `_headers` - Cloudflare Pages headers/CSP rules
- `robots.txt` - SEO robots file

### Optional (Can be Regenerated)
- **`build/`** - ❌ **DO NOT BACKUP** - This is generated by `npm run build` and can be recreated
- `node_modules/` - ❌ **DO NOT BACKUP** - Can be reinstalled with `npm install`

## Quick Backup Command

```bash
# Create a backup excluding build and node_modules
tar -czf backup-$(date +%Y%m%d).tar.gz \
  --exclude='build' \
  --exclude='node_modules' \
  --exclude='.git' \
  src/ public/ scripts/ package.json package-lock.json \
  tsconfig.json .gitignore _redirects _headers robots.txt
```

## What Gets Deployed

Only the `build/` folder is deployed to Cloudflare Pages. Everything else is source code that should be backed up.

## Recovery

If you need to restore:
1. Extract the backup
2. Run `npm install` to restore dependencies
3. Run `npm run build` to regenerate the `build/` folder
4. Deploy as normal
